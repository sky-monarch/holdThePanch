# Долгосрочное домашнее задание (игра "Hold The Panch")

## Этап I
Время на этап: **6969 часов**

### Цель

Необходимо разработать игру (2д платформер) для одного пользователя против компьютера. Правила игры можно найти ниже в ТЗ.

### Задание

На этом этапе необходимо реализовать desktop-приложение c графической отрисовкой процесса игры и реализацией основной логики игры.
#### Функционал, который должен быть реализован:
1. Игровое меню
    * кнопка **В БОЙ!** - запускает одиночную игру
    * кнопка **О создателях** - показывает экран с именами создателей игры;
    * кнопка **Настройки** - смена имен игроков, цвета танков, раскладки клавиатуры;
    * кнопка **ВЫХОД** - завершает работу приложения - с подтверждением
2. Игровой процесс
* визуальная часть:
    * отображение счётчика количества жизней в виде полоски здоровья над игроком
    * возможность поставить игру на паузу с отображением соответствующего уведомления
    * возможность продолжить игру после паузы с пропажей уведомления о паузе
    * возможность запустить игру заново, сделать рестарт уровня
    * по завершении игры последовательное выполнение следующих событий:
	* появление всплывающего уведомления об окончании игры
	* переход на следующий уровень в случае победы на всех уровнях, кроме последнего(предусмотреть не менее 5-ти уровней)
	* переход в главное меню в случае победы на последнем уровне или поражения
* логика игры:
  * Игрок начинает проходить игру с 1 уровня, главная задача Игрока убить всех врагов на уровне, чтобы перейти на следующий
  * На 5 уровне Игрок должен убить Босса чтобы пройти игру
  * Сложность игры увеличивается с каждым уровнем, увеличивается количесво врагов, их скорость, урон и количество здоровья
  * Каждый уровень появляются новые враги, количество их жизней также отображается сверху над ними
  * Когда у Игрока заканчиваются жизни, он проигрывает и может либо перейти в основное меню игры, либо перепройти уровень
  * В случае уничтожения Босса Игрок оповещается о победе и после переходит в основное меню
3. Конфигурация игры
    * все важные настройки игры хранить в файлах в любом удобном формате   


### Шаги разработки:

1. Время на работу: **69 часов**  

   * **Разработка Архитектуры игры, основных сцен и зависимостей**.  

        Необходимо разработать макет меню, который будет включать в себя все пункты, описанные в функционале,
        также необходимо разработать архитектуру игры, т.е определить основные сцены и зависимость между нимию
        Необходимо также проработать основые узлы, т.е составляющие сцен, после чего распределить обязанности в команде.

   * **Разработка основного меню игры**.  

     Необходимо разработать основное меню игры с кнопками: **В БОЙ!**,**О разработчиках** и **ВЫХОД** с указанным ранее функционалом.
     Для этого шага разработки **результатом** является готовое меню с фоном. Каждая кнопка должна открывать свой режим игры.

2. Время на работу: **69 часов**  

   * **Реализация Карт**  

    На этом шаге необходимо реализовать остальные карты, по тем же принципам и условиям описанным выше на прошлом этапе.

   * **Разработка меню выйгриша**  

    Необходимо реализовать сцену, которая будет оповещать и поздравлять с победой вслучае победы над Боссом и прохождении всех уровней.

   * **Добавление новых вграов и Босса**

    необходимо реализовать новых врагов и Босса по тем же принципам, свойствам и механикам, описанными выше на предыдущем этипе разработки.
    Необходимо в тоже время сделать каждого врага уникальным, т.е уникальный внешний вид и уникальные анимации, а также различные характеристики

   Для этого шага разработки **результатом** является полностью готовые модули для сборки игры на следующем этапе, все реализованные сцены отлажены в тестах
   и готовы к интеграции.

3. Время на работу: **69 часов**

   * **Интеграция сцен(Модулей)**

     необходимо реализовать полноценную игру, собрав все сцены и настроив их в дереве сцен, настроить порядок слежования сцен и взаимодействие между ними.

   * **тестирование сборки**  

    Необходимо провести тесты взаимодействия модулей между собой с целью устранения скрытых ошибок.

   * **Устранение ошибок**  

     Необходимо устранить ошибки выявленные в ходе Тестировании сборки

   Для этого шага разработки **результатом** является Реализованная и отлаженная одиночная игра, которая соответсвует всем определенным выше техническим требованиям.
